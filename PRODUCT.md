# Product

## Register

brand

## Platform

web

## Users

Claude Code users, mostly working developers, who land here from GitHub, a README, or a social link. Their job: understand in under a minute what these plugins do and get the install command into a terminal. Secondary audience: people evaluating Raj's craft through the work.

## Product Purpose

The marketing surface for the such-skills Claude Code plugin marketplace: four plugins and six skills built and used daily by one person. Success is a copied install command and a starred repo.

## Positioning

A one-person plugin registry where every plugin is something the author actually uses, and the page practices what its own no-slop plugin preaches.

## Conversion & proof

- Primary CTA: copy `claude plugin marketplace add rajnandan1/such-skills`. Secondary: the GitHub repo.
- The line a visitor remembers after 10 seconds: one command installs plugins someone actually uses.
- Belief ladder: (1) a plugin here does something I want; (2) it is real and maintained, with versions and changelogs; (3) installing costs one command.
- Proof on hand: live version numbers, per-plugin changelogs, MIT license, public GitHub history.

## Brand Personality

Loud, deadpan, handmade. The design shouts (shiba-orange drench, ink type, sticker borders); the words stay flat and factual. The doge name carries the joke so the copy never has to.

## Anti-references

The 2026 AI dev-tool landing page: near-black void, radial glows, grain overlay, gradient headline text, identical icon-card grids, uppercase eyebrow labels, animated stat counters. Also generic SaaS marketing voice ("Infinite possibilities", "Get started in seconds").

## Design Principles

- Color is the brand: one committed orange carries the page; ink does the talking.
- The command is the hero: the install line outranks any headline.
- Structure over decoration: skills live inside their plugin, tables and bands over card grids.
- Practice what ss-no-slop preaches: copy passes the plugin's own catalog.
- One loud entrance, then restraint: motion on load, calm on scroll.

## Accessibility & Inclusion

WCAG AA contrast throughout: ink on orange ≥ 4.5:1 for body text (no white-on-orange text). Full keyboard support for copy actions, visible focus states, `prefers-reduced-motion` honored, semantic landmarks and headings.
